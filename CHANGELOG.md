# Changelog

All notable changes to this project. Generated by scripts/release.sh.

## [Unreleased]

### Features
- feat(orders): part number is now required on every line; submitting with a blank one prompts a confirm dialog that offers an auto-derived value where possible (e.g. `MIXED_<capacity>_<interface>_<formFactor>` for Mixed-brand SSDs) or blocks otherwise. The synthetic value comes from a declarative rule registry in `@recycle-erp/shared`, backed by a server-side fallback at line creation; `Mixed` is now a selectable SSD brand (desktop + mobile submit)
- feat(scan): capture the in-app camera at high resolution and compress label scans client-side with Squoosh/MozJPEG before upload — closes the OCR clarity gap versus the native camera and keeps uploads well under the size cap
- feat(fe): register service worker on app boot (excludes vendor portal)
- feat(fe): link new PWA icon set + manifest from index.html
- feat(infra): caddy cache headers for sw.js, manifest, and workbox bundles
- feat(release): versioned Docker images + scripts/release.sh; runtime version on /api/health and in the mobile About sheet / desktop settings footer

### Refactors
- refactor(fe): DRY vendor-path check, log SW registration errors, simplify applyPwaUpdate return type

### Tooling
- chore(hooks): .githooks/ enforce a secret scan on commit and audit/changelog/static-security on push (enable with `git config core.hooksPath .githooks`)
