# Changelog

All notable changes to this project. Generated by scripts/release.sh.

## [Unreleased]

### Features
- feat(vendor): a single shareable "general" vendor link not tied to any customer — minted/copied/revoked from the Vendor links manager (rotates on regenerate). Bids from it arrive unattributed; the manager picks the customer when promoting the bid to a sell order, which back-fills it onto the bid (migration `0064`)
- feat(orders): part number is now required on every line; submitting with a blank one prompts a confirm dialog that offers an auto-derived value where possible (e.g. `MIXED_<capacity>_<interface>_<formFactor>` for Mixed-brand SSDs) or blocks otherwise. The synthetic value comes from a declarative rule registry in `@recycle-erp/shared`, backed by a server-side fallback at line creation, and its components are sanitized into a single token (quotes dropped, intra-component spaces folded to hyphens — `2.5"`→`2.5`, `M.2 2280`→`M.2-2280`); `Mixed` is now a selectable SSD brand (desktop + mobile submit)
- feat(scan): capture the in-app camera at high resolution and compress label scans client-side with Squoosh/MozJPEG before upload — closes the OCR clarity gap versus the native camera and keeps uploads well under the size cap
- feat(fe): register service worker on app boot (excludes vendor portal)
- feat(fe): link new PWA icon set + manifest from index.html
- feat(infra): caddy cache headers for sw.js, manifest, and workbox bundles
- feat(release): versioned Docker images + scripts/release.sh; runtime version on /api/health and in the mobile About sheet / desktop settings footer
- feat(pwa): polished mobile install onboarding — a frosted invitation card above the tab bar expands into an intro sheet with app benefits and platform-aware install guidance (one-tap native prompt on Android, illustrated Add-to-Home-Screen steps on iOS); EN + ZH strings, 14-day dismissal
- feat(orders): optional per-lot serial numbers on order lines (free-text blob, one SN per line) captured on submit and shown as compact monospace pills across order detail/review and inventory, on both desktop and mobile (migration `0063`)
- feat(vendor): faceted client-side catalog filtering in the vendor portal — search plus per-category attribute facets (RAM/SSD/HDD) with live counts, mirroring the desktop inventory schema

### Fixes
- fix(pwa): the mobile service worker no longer skip-waits on install, so a new version is never applied mid-session. It now stays in waiting until the user taps the update toast (`SKIP_WAITING` message → activate → reload), making the "Update available" prompt authoritative
- fix(analysis): the Inventory Analysis filters now scope every section. The page re-queries the server on each change so the warehouse filter also drives the per-type sub-analysis, status pipeline and top brands (previously only the KPI strip and category composition responded); value/sell/margin are now real at every scope. The page also moves under Inventory as an "Inventory ▸ Analysis" tab (route `/inventory/analysis`), replacing the standalone sidebar item
